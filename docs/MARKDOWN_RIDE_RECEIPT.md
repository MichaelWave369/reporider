# Markdown Ride Receipt Export

RepoRider can generate a copy-ready Markdown receipt after a mock create succeeds.

This export is the shareable counterpart to the on-screen Ride Complete summary.

## What it includes

The Markdown receipt includes:

- mode;
- repository URL;
- default branch;
- completion timestamp from the final receipt line;
- write-artifact count;
- approved starter-file count;
- approved starter-issue count;
- edited starter-file count;
- edited starter-issue count;
- file and issue draft character totals;
- queued starter file paths;
- queued starter issue titles;
- final receipt details;
- mock/live boundary note.

## Current implementation

The export is generated by `buildMarkdownRideReceipt(result)` from the same typed create result that powers the Ride Complete summary card.

That means the export should not drift from the on-screen result.

## Copy boundary

RepoRider currently renders the receipt in a read-only text box.

The app does not add a clipboard dependency yet. The rider copies the Markdown using normal device or browser select/copy controls.

This keeps the skeleton dependency-light while still proving the handoff workflow.

## Mock-mode boundary

In mock mode, the Markdown receipt does **not** prove that GitHub changed.

It proves that RepoRider:

1. built a plan;
2. collected file approvals;
3. collected issue approvals;
4. passed the safety boundary;
5. simulated repo creation;
6. produced a stable shareable ride receipt.

## Live-mode rule

When live GitHub writes land, the Markdown export should be generated from the real write result, not from UI assumptions.

The rider should be able to paste the exported receipt into:

- notes;
- a GitHub issue;
- a pull request;
- a project handoff;
- a support/debug report.
